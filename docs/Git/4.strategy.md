# 4. 브랜치 전략 심화

## 전략 비교
| 전략 | 주요 브랜치 | 특징 |
|------|-------------|------|
| Git Flow | `main`, `develop`, `feature/*`, `release/*` | 릴리스 단계 구분 명확 |
| GitHub Flow | `main`, `feature/*` | 단순, 빠른 개발 주기 |
| Trunk-Based | `main`만 사용 | CI/CD에 최적화 |

## 실무 적용 기준
- 소규모 스타트업 → GitHub Flow
- 릴리스 주기 명확한 제품팀 → Git Flow
- DevOps 팀 → Trunk-Based

## 실습
- 전략 하나 선택 → 역할 배분
- 브랜치 생성 → PR 흐름 진행
