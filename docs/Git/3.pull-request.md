# 3. 협업 흐름: Pull Request & 리뷰

## PR 개념
- **Pull**: 내 로컬로 가져오는 행위
- **Pull Request**: GitHub에서 협업자가 브랜치 병합 요청

## PR 흐름
1. 기능 브랜치 생성 → 커밋 → `push`
2. GitHub에서 Pull Request 생성
3. 팀원 리뷰 → 승인 또는 변경 요청
4. 병합 방식 결정 후 Merge

## Merge 방식
- **Merge**: 커밋 이력 유지
- **Squash**: 커밋 하나로 압축
- **Rebase**: 병합 히스토리를 간결하게 만듦

## 실습
- A/B 역할 분담 → 각각 브랜치 개발
- PR 요청 후 리뷰/코멘트 → 병합 → 충돌 상황 실습
